# デザインガイドライン (Design Guidelines)

このドキュメントは、今後の機能追加や改修において「Yorisol」のデザイン世界観を統一するための指針です。

## 1. デザインコンセプト
- **テーマ:** 「Yorisol (ヨリソル)」
- **キーワード:** 生きるに寄り添う、温かみ、信頼、プロフェッショナル。
- **基本方針:** 医療・介護現場で長時間見ても疲れない、優しく清潔感のあるUI。

## 2. カラーパレット (Strict Rules)
今後新しい画面やボタンを追加する際は、必ず以下の定義済みカラーを使用すること（ハードコード禁止）。

### Primary (アクション/強調)
- **クラス名:** `bg-primary`, `text-primary`
- **色味:** 温かみのあるオレンジ/アンバー系
- **使用箇所:** 
  - 「保存」「登録」などのポジティブなアクションボタン
  - アクティブなタブ
  - 強調したいアイコン

### Secondary (ベース/信頼)
- **クラス名:** `bg-secondary`, `text-secondary`
- **色味:** 深いネイビー/インディゴ系
- **使用箇所:** 
  - サイドバー背景
  - ヘッダー
  - 見出し文字（必要に応じて）
  - 重要な境界線

### Danger (危険)
- **クラス名:** `bg-red-600` (Tailwind Default), `text-destructive`
- **使用箇所:** 
  - 「削除」「キャンセル」などの破壊的操作
  - エラーメッセージ

### Background
- **クラス名:** `bg-background`
- **色味:** Slate-50 (薄いグレー)
- **使用箇所:** アプリケーション全体の背景（カードの背景は `bg-white`）

## 3. UIコンポーネント方針
- **角丸 (Border Radius):** 優しさを表現するため、ボタンやカードは `rounded-md` または `rounded-lg` で統一する。
- **余白 (Spacing):** 窮屈にならないよう、適切な `padding` (p-4, p-6) を確保する。
- **ボタン:** 原則として `components/ui/button.tsx` のコンポーネントを使用し、独自のスタイル定義は避ける。
