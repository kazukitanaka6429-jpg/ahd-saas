# デューデリジェンス(DD) 実施要領

このドキュメントは、AHD-SaaSプロジェクトの事業価値および技術的品質を定期的・定量的に評価するための標準手順書です。

## 1. 実施の目的
- 事業価値（Valuation）の定点観測
- 技術的負債の可視化と解消進捗の追跡
- 次期開発フェーズ（改善・機能追加）の意思決定支援

## 2. 実施タイミング（トリガー）
AIエージェントは以下のタイミングでユーザーにDDの実施を提案してください。
1. **主要機能のリリース完了時**（例: 請求書自動生成、AI配置最適化など）
2. **大規模なリファクタリング完了時**（例: テストカバレッジが目標値に達した、RLSの完全移行など）
3. **四半期ごとの定点観測**（3ヶ月に1回）
4. **事業フェーズの変化時**（例: 外部顧客導入決定、30施設展開開始など）

## 3. 評価手法（Methodology）

評価は必ず以下の構成で実施し、前回結果との差分（Diff）を明記すること。

### 3.1 技術デュージュリジェンス (Tech DD)

#### A. コードベース分析
以下のコマンド等を用いて定量データを取得する。
- 総行数（LOC）
- ファイル数、コンポーネント数
- 依存パッケージ数（package.json）
- マイグレーションファイル数

#### B. 品質スコアリング (10点満点)
以下の7項目を各10点満点で採点し、平均を算出する。
1. **機能完成度**: 要件に対する実装率。バグの少なさ。
2. **コード整理**: 可読性、ディレクトリ構造、命名規則。
3. **テスト**: カバレッジ率、主要ロジックのテスト有無。
4. **セキュリティ**: RLSポリシー、認証・認可、データ保護。
5. **ドキュメント**: 設計書、API仕様、セットアップ手順の充実度。
6. **デプロイメント**: CI/CD、IaC、環境分離。
7. **パフォーマンス**: レスポンス速度、バンドルサイズ、クエリ効率。

### 3.2 事業価値算定 (Valuation)

以下の3手法を用いて算出し、加重平均で最終評価額を決定する。

#### 手法①: コスト積上法 (Cost Approach)
「同等のシステムをゼロから再構築した場合のコスト」を算出する。
- **計算式**: `(開発工数(人日) × 単価) + (設計・管理工数 × 単価) - 技術的負債割引`
- **単価設定**: 開発 ¥50,000/日、上流 ¥70,000/日 などを基準とする。

#### 手法②: 収益還元法 (Income Approach / DCF)
「将来生み出すキャッシュフローの割引現在価値」を算出する。
- **内部利用（自社30施設）の扱い**: コスト削減効果を「確定収益」とみなして算入する。
- **外部販売**: 実現確率で割り引いて加算する。
- **計算式**: `Σ(各年のFCF / (1 + 割引率)^n) + ターミナルバリュー`

#### 手法③: 市場比較法 (Market Approach)
「類似SaaS企業の評価倍率（マルチプル）」を適用する。
- **計算式**: `ARR (Annual Recurring Revenue) × マルチプル倍率`
- **マルチプル基準**: 
    - 国内SaaS中央値: 8〜12x
    - アーリーステージ/未上場: 3〜5x
    - 介護・福祉特化: 5〜8x

## 4. 成果物フォーマット

レポートは以下のパスに保存する。
`docs/due_diligence_valuation/reports/YYYY-MM-DD_{context}.md`

レポートには必ず以下のセクションを含めること。
1. **エグゼクティブサマリー**: 評価額、品質スコア、前回比。
2. **Top 5 改善提言**: 価値向上に最も寄与する具体的なアクション5選（影響金額付き）。
3. **詳細分析**: Tech DDおよびValuationの詳細計算過程。
